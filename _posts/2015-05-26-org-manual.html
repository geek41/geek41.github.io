org manual 整理.

1 Publishing

1.1 org-publish-project-alist变量

通过为每一个变量配置来定义发布，成为org-publish-project-alist

1.2 源文件及发布目录

大多数属性都是可选的，但有些应始终设置。 特别是，org需要知道到哪里寻找源文件，并放在哪里发布文件。

:base-directory

含目录发布源文件

:publishing-directory

目录下的输出文件将予以公布。 您可以使用适合于Emacs的流浪汉包文件名 ​​语法直接发布到Web服务器。 或者你可以发布到本地目录，并用它来 ​​上传您的网站（见外部工具上传文件 ）。

:preparation-function

功能或功能列表中被称为起始发布过程，例如之前，要运行make的更新文件被出版。 该项目的属性列表的范围限定到这个电话作为变量project-plist 。

:completion-function

函数或函数调用整理出版过程，例如后，改变结果文件的权限列表。 该项目的属性列表的范围限定到这个电话作为变量project-plist 。

2 导出Freemind 思维导图

emacs官方内置的org不再包含contributed libraries,如果你想用到他们,可以从github获取,链接:https://github.com/emacsmirror/org ,然后将他们加入到 load-path. 

(add-to-list 'load-path "~/path/to/orgdir/contrib/lisp" t)

在.emacs 中加入 '(require 'ox-freemind)'

3 用plantUML画uml图

PlantUML是一个快速创建UML图形的组件，官网上之所以称它是一个组件，我 想主要是因为多数情况下我们都是在Eclipse、NetBenas、Intellijidea、 Emacs、Word等软件里来使用PlantUML（参看各软件相关配置）。

PlantUML支持的图形有：

  • sequence diagram,
  • use case diagram,
  • class diagram,
  • activity diagram (here is the new syntax),
  • component diagram,
  • state diagram,
  • object diagram,
  • wireframe graphical interface

PlantUML通过简单和直观的语言来定义图形，它可以生成PNG、SVG和二进制 图片。

3.1 在Emacs里配置PlantUML

1.下载 plantuml.jar 到你的硬盘上（官网下载页面） 2.安装生成图片用的软件Graphviz 3.在 .emacs 里添加Org-mode配置，把 plantuml 添加到 org-babel-load-languages 里。 

;; active Org-babel languages
(org-babel-do-load-languages
 'org-babel-load-languages
 '(;; other Babel languages
   (plantuml . t)))

然后把刚下载到的 plantuml.jar 文件的存放路径也添加到 .emacs 文件中。 

(setq org-plantuml-jar-path
      (expand-file-name "~/path/to/plantuml.jar"))

下面是一个简单的示例： 

#+BEGIN_SRC plantuml :file ../img/orgmode-babel-sequenceuml.png
  Alice -> Bob: synchronous call
  Alice ->> Bob: asynchronous call
#+END_SRC
